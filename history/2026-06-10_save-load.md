# 2026-06-10: Save/Load (spec 020)

## What was done
- SaveSystem with localStorage autosave (AIMING/SHOP transitions), terrain
  PNG-dataURL serialization, AI controller revival, market restore, and a
  Continue Saved Game button on the setup screen.

## Lessons
- TerrainSystem keeps three derived structures (mask, heightMap,
  terrainPixels) — all rebuildable from the canvas, so a PNG data URL is the
  whole terrain snapshot (~tens of KB, well within localStorage limits).
  Rebuild mask with the same alpha>20 threshold as PNG map loading.
- Save only in stable phases; projectiles/explosions are not resumable and
  are dropped on load.
- Border strategy is engine-side (not in GameState), so loading must re-apply
  it from state.borderMode — extracted applyBorderMode() in GameEngine.
- Phase-transition detection at the end of GameEngine.update() is a reliable
  single hook for autosave (transitions happen in several systems).
